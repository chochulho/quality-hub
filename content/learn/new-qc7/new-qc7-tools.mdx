## 신 QC 7가지 도구란?

신(新) QC 7가지 도구(New QC Seven Tools)는 1977년 일본과학기술연맹(JUSE)이 정립한 도구로, 기존 QC 7가지 도구가 다루기 어려웠던 **정성적(언어적) 데이터와 기획 단계 문제**를 구조화합니다. 주로 **관리자·기술자·기획자** 계층에서 활용합니다.

## QC 7도구 vs 신 QC 7도구

| 구분 | QC 7가지 도구 | 신 QC 7가지 도구 |
|---|---|---|
| 데이터 유형 | 정량적 (수치) | 정성적 (언어·아이디어) |
| 주요 단계 | 현장 분석·통제 | 기획·설계·방침 관리 |
| 주 사용자 | 현장 작업자·품질 | 관리자·기술자·기획자 |

## 7가지 도구 한눈에 보기

| # | 도구 | 목적 | 핵심 산출물 |
|---|---|---|---|
| 1 | **친화도** | 아이디어 그룹화 | 유사 항목 분류 그룹 |
| 2 | **연관도** | 원인-결과 관계 파악 | 핵심 원인 순위 |
| 3 | **계통도** | 목표-수단 전개 | 실행 수단 계층 구조 |
| 4 | **매트릭스도** | 요소 간 관계 평가 | 가중 우선순위 행렬 |
| 5 | **매트릭스 데이터 해석** | 다변수 상관 파악 | 상관계수 히트맵 |
| 6 | **PDPC** | 리스크 대응 계획 | 위험-대책 플로우 |
| 7 | **애로우 다이어그램** | 일정 계획·임계경로 | 간트 차트 + CPM |

## ① 친화도 (Affinity Diagram / KJ법)

친화도는 **대량의 언어 데이터를 유사성 기준으로 그룹화**하는 도구입니다. 일본 인류학자 가와키타 지로(川喜田二郎)가 개발하여 KJ법이라고도 합니다.

**절차**: 브레인스토밍 → 카드 1장에 아이디어 1개 → 침묵 속에서 유사 카드 그룹화 → 그룹 이름 부여

- 팀 합의·VOC 분석·복잡한 문제 발굴에 효과적
- 논리보다 **직관적 유사성**으로 먼저 분류

## ② 연관도 (Relations Diagram)

연관도는 **복잡하게 얽힌 요인들의 원인-결과 관계를 화살표로 연결**하여 핵심 원인을 찾는 도구입니다.

- **출력 화살표 수(원인) > 입력 화살표 수(결과)** → 핵심 원인 요인
- 순환 관계도 표현 가능 (인과 루프)
- 특성요인도와 달리 계층 구조 없이 자유로운 관계 표현

```
[원자재 품질 불안정] ──→ [불량률 증가] ──→ [고객 불만]
        ↑                      ↑
[공급업체 관리 미흡] ──→ [입고 검사 생략]
```

## ③ 계통도 (Tree Diagram / Systematic Diagram)

계통도는 **목표를 달성하기 위한 수단을 계층적으로 전개**하는 도구입니다. 목표→1차 수단→2차 수단→3차 수단으로 세분화합니다.

- 방침 관리의 목표 전개, FMEA 대책 수립에 활용
- 가장 하위 수단이 실제 실행 가능한 행동이어야 함
- 친화도 그룹 → 계통도 수단 전개 순서로 연계

## ④ 매트릭스도 (Matrix Diagram)

매트릭스도는 **두 요소 그룹의 관계 강도를 기호로 표시**하는 행렬 도구입니다.

| 기호 | 의미 | 점수 |
|---|---|---|
| ◎ | 강한 관계 | 9점 |
| ○ | 중간 관계 | 3점 |
| △ | 약한 관계 | 1점 |
| (빈칸) | 관계 없음 | 0점 |

열(수단) 합산 점수로 **우선순위 자동 계산**. L형(2요소), T형(3요소), X형(4요소) 행렬이 있으며, QFD(품질기능전개)의 품질의 집이 대표적인 L형 매트릭스도.

## ⑤ 매트릭스 데이터 해석 (Matrix Data Analysis)

매트릭스 데이터 해석은 **여러 변수 간 상관계수 행렬을 계산하여 변수 간 관계를 히트맵으로 시각화**합니다. 신 QC 7도구 중 유일하게 수치 계산이 필요한 도구입니다.

- Pearson 상관계수(r)로 -1 ~ +1 사이 값
- r → +1(빨강): 강한 양의 상관, r → -1(파랑): 강한 음의 상관
- 주성분분석(PCA)으로 변수를 2~3차원으로 축소하기도 함

## ⑥ PDPC (Process Decision Program Chart)

PDPC는 **계획 실행 중 예상되는 위험과 대책을 사전에 플로우차트로 도식화**합니다. "만약 X가 발생하면 Y로 대응한다"는 컨틴전시 플랜 시각화 도구입니다.

```
[단계 1] → [단계 2] → [단계 3] → 목표 달성
              ↓
           [위험 A]
              ↓
           [대책 A'] → 복귀
```

- 신제품 개발·프로젝트 리스크 관리에 활용
- FMEA와 달리 **계획의 흐름** 속에서 위험을 파악

## ⑦ 애로우 다이어그램 (Arrow Diagram / PERT·CPM)

애로우 다이어그램은 **작업(Activity)의 선후 관계를 화살표로 연결하고 임계 경로(Critical Path)를 계산**하는 일정 관리 도구입니다.

**CPM 계산**:
```
ES(최조 시작) = max(선행 작업들의 EF)
EF(최조 완료) = ES + Duration
LS(최지 시작) = LF - Duration
LF(최지 완료) = min(후행 작업들의 LS)
여유(Float)   = LS - ES  (= 0이면 임계 경로)
```

- **임계 경로(Critical Path)**: Float = 0인 작업들의 연속 — 이 경로 지연 시 전체 일정 지연
- 프로젝트 관리(PMP), 양산 이관(PPAP) 일정 관리에 활용

## 도구 선택 가이드

```
문제 상황
├── 아이디어가 너무 많아 정리가 안 됨 → 친화도
├── 원인이 복잡하게 얽혀 있음 → 연관도
├── 목표는 있는데 실행 방법이 막막함 → 계통도
├── 여러 수단 중 무엇을 먼저 할지 모름 → 매트릭스도
├── 변수 간 어떤 관계가 있는지 알고 싶음 → 매트릭스 데이터 해석
├── 계획 진행 중 문제 발생 시 대응 준비 → PDPC
└── 프로젝트 일정을 체계적으로 관리해야 함 → 애로우 다이어그램
```

## IATF 16949 연관 조항

- **6.1**: 리스크와 기회 다루기 — PDPC, 연관도
- **8.3**: 제품·서비스 설계 개발 — 계통도, 매트릭스도 (QFD)
- **9.3**: 경영 검토 — 친화도, 매트릭스도로 개선 우선순위 도출

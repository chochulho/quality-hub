## QFD란?

품질기능전개(Quality Function Deployment, QFD)는 **고객의 요구사항(Voice of Customer, VOC)을 설계·부품·공정 매개변수로 단계적으로 변환**하는 구조화된 방법론입니다.

- 개발: Yoji Akao (1966, 일본)
- 목적: "고객이 원하는 것"을 "기술자가 만들어야 하는 것"으로 체계적 번역
- IATF 16949 8.3.3 설계 및 개발 입력의 핵심 도구

## 4단계 폭포 구조 (Cascade)

```
Phase 1: 제품 계획
  WHATs = 고객 요구사항 (VOC)
  HOWs  = 제품 특성 (Engineering Characteristics)
        ↓ 이전 (Phase 1 HOWs → Phase 2 WHATs)

Phase 2: 부품 전개
  WHATs = 제품 특성
  HOWs  = 부품 특성 (Part Characteristics)
        ↓ 이전

Phase 3: 공정 계획
  WHATs = 부품 특성
  HOWs  = 공정 매개변수 (Process Parameters)
        ↓ 이전

Phase 4: 생산 계획 (선택)
  WHATs = 공정 매개변수
  HOWs  = 생산 방법 (Production Controls)
```

각 단계의 결과물(HOWs)이 다음 단계의 입력(WHATs)이 되어 **고객 요구가 공정 수준까지 추적 가능**합니다.

## 품질의 집 (House of Quality, HOQ) 구조

HOQ는 QFD Phase 1의 핵심 도구로 7가지 구성 요소를 갖습니다.

| 번호 | 구성 요소 | 설명 |
|---|---|---|
| ① | WHATs (고객 요구사항) | VOC 목록, 좌측 열 |
| ② | 중요도 (Importance) | 고객 관점 중요도, 1~5 또는 1~10 점수 |
| ③ | HOWs (기술 특성) | 설계 파라미터, 상단 열 |
| ④ | 관계 매트릭스 (Body) | WHATs × HOWs 상관 강도 |
| ⑤ | 지붕 (Roof) | HOWs 간 기술 상관 (상삼각 행렬) |
| ⑥ | 가중 점수 (Weighted Score) | Σ(중요도 × 강도), 하단 행 |
| ⑦ | 목표값 (Target) | 각 HOW의 설계 목표 |

### 관계 강도 기호

| 기호 | 의미 | 점수 |
|---|---|---|
| ◎ | 강한 관계 | 9 |
| ○ | 보통 관계 | 3 |
| △ | 약한 관계 | 1 |
| (빈칸) | 관계 없음 | 0 |

### 지붕 상관 기호

| 기호 | 의미 |
|---|---|
| ++ | 강한 양의 상관 (함께 좋아짐) |
| + | 약한 양의 상관 |
| - | 약한 음의 상관 (trade-off) |
| -- | 강한 음의 상관 |

## 가중 점수 계산

```
가중 점수(HOW_j) = Σ [중요도(WHAT_i) × 강도(WHAT_i, HOW_j)]
```

**예시**: VOC 3개, 설계 특성 4개

| VOC | 중요도 | 내구성(H1) | 경량화(H2) | 방수(H3) | 소음(H4) |
|---|---|---|---|---|---|
| 장기간 사용 | 5 | ◎(9) | ○(3) | ◎(9) | △(1) |
| 가볍게 휴대 | 4 | △(1) | ◎(9) | ○(3) | ○(3) |
| 조용한 작동 | 3 | ○(3) | △(1) | | ◎(9) |

```
H1 가중점수 = 5×9 + 4×1 + 3×3 = 45 + 4 + 9 = 58
H2 가중점수 = 5×3 + 4×9 + 3×1 = 15 + 36 + 3 = 54
H3 가중점수 = 5×9 + 4×3 + 3×0 = 45 + 12 + 0 = 57
H4 가중점수 = 5×1 + 4×3 + 3×9 = 5 + 12 + 27 = 44
```

→ 설계 우선순위: **내구성(58) > 방수(57) > 경량화(54) > 소음(44)**

## QFD 작성 6단계

1. **VOC 수집**: 고객 인터뷰·설문·클레임 분석으로 요구사항 목록화
2. **중요도 결정**: AHP·쌍비교·설문으로 각 VOC 중요도 점수 부여
3. **HOW 도출**: 브레인스토밍·기술 검토로 충족 방법(기술 특성) 정의
4. **관계 매트릭스 작성**: 각 WHATs × HOWs 셀에 강도 기호 입력
5. **지붕 상관 분석**: HOWs 간 기술적 trade-off / synergy 파악
6. **우선순위 결정**: 가중 점수로 HOWs 순위 → 설계 목표값 설정

## IATF 16949 연관

QFD는 IATF 16949 8.3.3 *설계 및 개발 입력* 요구사항을 체계적으로 충족합니다.

| IATF 요구사항 | QFD 대응 |
|---|---|
| 고객 요구사항 반영 | Phase 1 WHATs = VOC |
| 규제·법규 요구사항 | VOC 항목에 포함 |
| 리스크 분석 | 지붕의 trade-off 파악 |
| 이전 설계 정보 활용 | HOW 목표값에 반영 |

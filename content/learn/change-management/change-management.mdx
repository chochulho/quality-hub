## 변경관리란?

**변경관리(Change Management)**는 생산·서비스 제공 조건의 변경이 품질에 미치는 영향을 사전에 평가하고, 승인된 절차에 따라 통제하는 시스템입니다.

IATF 16949 **8.5.6 생산 및 서비스 제공 변경 관리** 조항은 변경 발생 시 품질 영향 평가, 내부 승인, 고객 통보, 기록 유지를 의무화합니다.

### 변경관리가 필요한 이유

```
변경 전 상태로 PPAP/양산 승인 → 변경 발생
→ 미통보·미승인 변경은 IATF 심사 주요 부적합 사항
→ 고객 클레임, 리콜, 계약 위반으로 이어질 수 있음
```

---

## 1. 4M 변경 유형

생산에 영향을 주는 4가지 자원을 기준으로 변경을 분류합니다.

| 분류 | 영문 | 대표 예시 |
|---|---|---|
| **Man** | 사람 | 신규 작업자 배치, 담당자 교체, 교번 변경, 외주화 |
| **Machine** | 설비 | 설비 교체·추가, PM 주기 변경, 금형 교체, 지그 수정 |
| **Material** | 재료 | 원자재 사양 변경, 공급업체 변경, 부품 번호 개정 |
| **Method** | 방법 | 작업 표준서 개정, 공정 순서 변경, 파라미터(온도·속도·압력) 변경 |

> **4M + E (Environment)**: 일부 고객은 환경(작업 환경 이전, 생산 라인 이동) 변경도 별도 관리 요구.

### 경미한 변경과 주요 변경의 구분

| 구분 | 기준 | 대응 |
|---|---|---|
| **주요 변경** | 제품 특성·PPAP 기준에 영향 | 고객 승인 또는 통보 필요 |
| **경미한 변경** | 내부 기준 범위 내 조정 | 내부 승인으로 완결 |

경계는 고객 요구사항(PSW, PPAP Manual)에 따라 다르므로 반드시 확인 필요.

## 2. 변경 위험도 분류

변경이 제품 품질에 미치는 잠재적 영향을 사전에 평가합니다.

| 위험도 | 판단 기준 | 승인 절차 |
|---|---|---|
| **고 (High)** | 안전·법규 특성 영향, PPAP 기준 변경, 공급업체 변경 | 고객 공식 승인 必 + PPAP 재제출 검토 |
| **중 (Medium)** | 공정 파라미터 변경, 설비 교체 | 내부 승인 후 고객 통보 |
| **저 (Low)** | 동일 사양 내 경미한 조정 | 내부 승인으로 완결, 기록 유지 |

---

## 3. 4단계 변경 승인 프로세스

IATF 16949 8.5.6 기반 표준 프로세스입니다. 스크린샷의 4M Change Manager가 이 4단계를 그대로 구현합니다.

```
P1 제안 (Proposal)
  → 변경 내용 기술, 4M 분류, 영향 평가, 위험도 결정
  → 산출물: Change Request(CR) 등록

P2 검토 (Review)
  → 관련 부서 검토, 기술 검증, 실행 계획 수립
  → FMEA·Control Plan 업데이트 필요 여부 확인
  → 산출물: 검토 의견, 승인·반려 결정

P3 승인 (Approval)
  → 내부 승인 (품질·생산·엔지니어링)
  → 고객 통보 (Customer Notification)
  → 고객 공식 승인 획득 (위험도 高인 경우)
  → 산출물: 승인 서명, EO(Engineering Order) 발행

P4 유효성 평가 (Validation)
  → 변경 실행 (파일럿 런 또는 초도 양산)
  → 검증 데이터 수집 (치수, 기능, 공정능력)
  → 효과 확인 후 종결
  → 산출물: 검증 보고서, 기록 보관
```

## 4. 고객 통보 요건

### 언제 통보해야 하나?

PPAP(Production Part Approval Process) Manual 및 고객 요구사항에 따라 다르지만, 일반적으로 다음 변경은 고객 통보·승인이 필요합니다.

| 변경 유형 | 일반적 요건 |
|---|---|
| 원자재 공급업체 변경 | 고객 승인 필수 |
| 제조 공장 이전 | 고객 승인 필수 |
| 서브 공급업체 변경 (Tier 2) | 고객 통보 (승인은 고객사별 상이) |
| 설비 교체 (동일 사양) | 내부 승인 후 통보 |
| 작업 표준 개정 (파라미터 범위 내) | 내부 승인으로 완결 |

> 기준은 **고객별 특정 요구사항(CSR: Customer Specific Requirements)**이 우선합니다.

### PPAP 재제출 조건

다음 변경은 Level 3 이상 PPAP 재제출이 필요할 수 있습니다.

```
• 설계 변경 (엔지니어링 변경 = EO 발행)
• 새로운 공급업체의 원자재 사용
• 생산 공장 또는 라인 이전
• 생산 방법 또는 공정 순서의 중대한 변경
• 새로운 또는 수정된 금형·다이·주형 사용
• 생산 중단 후 12개월 이상 경과 후 재개
```

## 5. EO(Engineering Order)와 CR(Change Request) 관계

```
변경 제안
    ↓
CR (Change Request) 개설 — 내부 변경 관리 번호
    ↓
기술 검토 → 설계 변경 필요 시
    ↓
EO (Engineering Order) 발행 — 고객 또는 내부 설계 변경 번호
    ↓
EO 번호로 PPAP·도면·BOM 업데이트
    ↓
고객에게 EO 번호 포함 통보
```

**CR**: 내부 변경 요청 추적 번호 (예: CR-2024-0124)  
**EO**: 설계 변경이 포함된 경우 발행하는 공학적 변경 지시서 (예: EO-2024-0445)

## 6. 변경 기록 관리 요구사항

IATF 16949 10.2.2 및 8.5.6 기반:

| 기록 항목 | 내용 | 보유 기간 |
|---|---|---|
| 변경 요청서 (CR) | 변경 내용, 위험도, 승인자 | 제품 수명 + α (고객 요구사항 따름) |
| 고객 통보 이력 | 통보 일자, 내용, 고객 응답 | 동일 |
| 검증 결과 | 초도품 성적서, 공정능력 데이터 | 동일 |
| EO 문서 | 설계 변경 내용, EO 번호 | 동일 |

> IATF 심사 시 변경관리 기록은 **증적(Evidence)** 으로 즉시 제시해야 합니다.

## 7. IATF 16949 8.5.6 조항 핵심 내용

```
8.5.6 생산 및 서비스 제공 변경 관리

조직은 생산 또는 서비스 제공 변경을 위해:
• 변경 전 품질에 대한 영향을 평가할 것
• 내부적으로 변경을 승인할 것
• 필요한 경우 고객 및 규제 기관의 승인을 획득할 것
• 기록을 보유할 것 (변경 내용, 승인자, 실행 일자)

조직은 변경의 영향을 검증하고 모니터링해야 한다.
필요한 경우 PPAP를 재제출해야 한다.
```

## 품질기술사 시험 출제 포인트

### 자주 나오는 문제 유형

- **4M 변경의 4가지 유형을 설명하고 각각의 예를 드시오**
- **IATF 16949 8.5.6 변경관리 요구사항을 서술하시오**
- **고객 통보가 필요한 변경과 내부 승인으로 완결할 수 있는 변경을 구분하여 설명하시오**
- **PPAP 재제출이 필요한 변경의 유형을 나열하시오**
- **EO와 CR의 차이를 설명하시오**

### 암기 키워드

```
4M = Man·Machine·Material·Method (+ E: Environment)
변경 4단계 = 제안(P1) → 검토(P2) → 승인(P3) → 유효성평가(P4)
IATF 8.5.6 = 영향평가 + 내부승인 + 고객승인 + 기록보유 + PPAP 재제출 검토
CR = 내부 변경 요청 번호
EO = 설계 변경 지시서 (고객·내부 설계 변경 포함)
```

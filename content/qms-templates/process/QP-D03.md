## 1. 목적

PFMEA(Process Failure Mode and Effects Analysis)를 통해 공정 설계 단계에서 잠재적 고장을 식별하고, AP(Action Priority) 기반으로 우선순위를 정해 예방 조치를 실시한다.

## 2. 적용 범위

신규 공정 개발, 공정 변경, 반복 불량 발생 시 전체 PFMEA 재검토에 적용한다.  
{{cert_target_iatf}} 조항 8.3.3, 8.5.1, 9.2 요건과 연계한다.

## 3. 방법론 — AIAG-VDA 7단계 (2019)

> RPN(위험우선지수) 방식은 사용하지 않는다. AP(Action Priority) 방식을 적용한다.

### Step 1: 계획 및 준비

- PFMEA 범위 정의: 공정 흐름도 기준
- 팀 구성: {{dept_engineering}}, {{dept_quality}}, {{dept_production}}
- 기준선 FMEA 또는 유사 공정 FMEA 참조

### Step 2: 구조 분석 (Structure Analysis)

공정을 3계층으로 분해:

```
프로세스 항목 (공정 단계)
  └─ 공정 단계 (세부 작업)
       └─ 공정 특성 (작업 요소)
```

산출물: **P-Diagram** (프로세스 다이어그램)

### Step 3: 기능 분석 (Function Analysis)

각 구조 요소의 기능·요구사항 정의:
- 공정 단계의 **의도된 기능** 명시
- 특별특성(SC) 연결

### Step 4: 고장 분석 (Failure Analysis)

고장 체인: **고장 영향(FE)** → **고장 모드(FM)** → **고장 원인(FC)**

- **고장 영향**: 고객(최종 사용자, 제조 공장)에 미치는 영향
- **고장 모드**: 공정 요건이 충족되지 않는 방식
- **고장 원인**: 고장 모드의 발생 원인 (특정·통제 가능해야 함)

### Step 5: 리스크 분석 (Risk Analysis)

| 평가 항목 | 척도 | 설명 |
|---------|------|------|
| **S (Severity)** | 1~10 | 고장 영향의 심각성 |
| **O (Occurrence)** | 1~10 | 고장 원인의 발생 가능성 |
| **D (Detection)** | 1~10 | 고장 감지 가능성 (낮을수록 감지 잘 됨) |

**AP (Action Priority) 판정표**:

| S | O | D | AP |
|---|---|---|---|
| 9~10 | 모든 값 | 모든 값 | **H (High)** — 즉시 조치 |
| 5~8 | 4~10 | 7~10 | **H** |
| 5~8 | 4~10 | 1~6 | **M (Medium)** |
| 1~4 | 1~3 | 1~6 | **L (Low)** |

- **AP=H**: 반드시 조치, 완료일 지정
- **AP=M**: 팀 검토 후 조치 여부 결정
- **AP=L**: 수용 가능, 기회 있을 때 개선

### Step 6: 최적화 (Optimization)

1. AP=H 항목부터 예방·감지 조치 수립
2. 조치 완료 후 S/O/D 재평가 → AP 재판정
3. **조치 효과 검증**: 실제 공정 데이터로 O, D 재평가

### Step 7: 결과 문서화

- PFMEA 최종 검토 및 승인 ({{dept_quality}} 승인)
- Control Plan과 1:1 연계 확인
- 특별특성은 반드시 Control Plan에 반영

---

## 4. PFMEA-Control Plan 연계

```
PFMEA 고장 원인
  → 예방 관리 방법 → Control Plan "방법" 열
  → 감지 관리 방법 → Control Plan "검사 방법" 열
  → 특별특성(SC)  → Control Plan "특별특성" 표기
```

특별특성이 있는 공정 단계는 **Control Plan에 반드시 포함**하고, SPC 또는 100% 검사 적용 여부를 명시한다.

## 5. PFMEA 갱신 트리거

| 사유 | 갱신 범위 | 기한 |
|------|---------|------|
| 신규 공정 개발 | 전체 작성 | PPAP 전 |
| 공정 변경 (4M) | 변경 공정 단계 | 변경 후 즉시 |
| 고객 클레임 | 해당 공정 단계 + 유사 공정 | 8D 완결 전 |
| 내부 심사 지적 | 해당 항목 | NCR 조치 기한 내 |
| 반복 불량 (3회 이상) | 해당 공정 전면 재검토 | 30일 내 |

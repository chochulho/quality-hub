## 1. 목적

PFMEA(Process Failure Mode and Effects Analysis)를 AIAG-VDA 4th Edition 7단계 방법론에 따라 체계적으로 작성·유지하여 공정 실패 모드를 사전에 파악하고, 관리 계획서(Control Plan) 및 작업지침서(WI)와 연계함으로써 공정 품질을 확보한다.

---

## 2. 적용 범위

다음 모든 경우에 PFMEA 작성 또는 갱신을 의무 적용한다.

| 적용 사유 | PFMEA 범위 | 기한 |
|---|---|---|
| 신규 공정 개발 (APQP Phase 3) | 전체 신규 작성 | PPAP 제출 전 |
| 공정 변경 — 4M (사람·설비·재료·방법) | 변경 공정 단계 + 영향 공정 | 변경 적용 전 |
| 고객 클레임 또는 내부 불량 (반복 3회 이상) | 해당 공정 + 유사 공정 재검토 | 8D 완결 전 |
| 내부 심사 또는 고객 심사 지적 | 해당 항목 | NCR 조치 기한 내 |
| 정기 검토 | 전체 PFMEA | 연 1회 이상 |

## 5. 세부 절차 — AIAG-VDA 7단계 방법론

### P1. PFMEA 착수 (Step 1: Planning & Preparation)

**담당**: {{dept_quality}} + {{dept_engineering}}  **기한**: 공정 설계 초기 (APQP Phase 3 착수 시)

1. **프로젝트 범위 정의**: 공정 흐름도 기반 적용 공정 단계 확정
2. **다기능 팀(CFT) 구성**: 품질·생산·개발·구매 — 최소 3개 부서 이상 참여 필수
3. **기준선 PFMEA 선택**: 유사 제품·공정의 기존 PFMEA 또는 참조 PFMEA 식별
4. **고객 PFMEA 양식·요건 확인**: 고객 CSR에 특정 양식·등급 기준이 있는 경우 우선 적용
5. **킥오프 미팅**: 범위·일정·담당 확인, 기준선 PFMEA 브리핑

---

### P2. 구조 분석 (Step 2: Structure Analysis)

**담당**: 다기능 팀  **기한**: 공정 흐름도 확정 후

공정을 3계층으로 분해하여 P-Diagram(프로세스 다이어그램) 작성:

```
프로세스 항목 (전체 제조 공정)
  └─ 공정 단계 (예: 가공 → 조립 → 검사)
       └─ 공정 요소 (4M: 사람·기계·자재·방법)
```

1. 공정 흐름도의 각 공정 번호와 PFMEA 행 1:1 매칭
2. **4M(Man·Machine·Material·Method)** 기반 공정 요소 분류
3. 구조 분석 결과를 PFMEA 양식의 "공정 단계" 열에 기입

---

### P3. 기능·고장 분석 (Step 3 & 4: Function Analysis & Failure Analysis)

**담당**: 다기능 팀

**기능 분석**:
1. 각 공정 요소의 **의도된 기능(Function)** 및 요구 사항(Requirement) 정의
2. 특별 특성(SC/CC)이 있는 공정 요소: 기능 정의 시 특별 특성 명시적 연결

**고장 분석 — 고장 체인 도출**:

```
고장 영향(Failure Effect, FE)
  → 고장 모드(Failure Mode, FM)
    → 고장 원인(Failure Cause, FC)
```

| 항목 | 정의 | 예시 |
|---|---|---|
| **고장 영향 (FE)** | 고객(최종 사용자·제조 공장)에 미치는 영향 | 기능 불량, 안전 위험, 조립 불가 |
| **고장 모드 (FM)** | 공정 요건이 충족되지 않는 방식 | 치수 초과, 표면 스크래치, 누락 조립 |
| **고장 원인 (FC)** | 고장 모드의 근본 원인 (특정·통제 가능해야 함) | 공구 마모, 작업자 누락, 재료 편차 |

---

### P4. 위험 분석 (Step 5: Risk Analysis)

**담당**: 다기능 팀  **기한**: 고장 분석 완료 후

**S·O·D 평가 기준**:

| 평가 항목 | 척도 | 의미 |
|---|---|---|
| **S (Severity, 심각도)** | 1~10 | 고장 영향의 심각성 (10: 안전·법규 위반) |
| **O (Occurrence, 발생도)** | 1~10 | 고장 원인의 발생 가능성 (10: 매우 높음) |
| **D (Detection, 검출도)** | 1~10 | 현행 관리 방법으로의 감지 가능성 (10: 감지 거의 불가) |

> AIAG-VDA 방식: RPN(S×O×D) 산출 금지. AP(Action Priority) 기반 우선순위 결정.

**AP(Action Priority) 판정표**:

| S | O | D | AP 등급 |
|---|---|---|---|
| 9~10 | 모든 값 | 모든 값 | **H (High)** — 즉시 조치 필수 |
| 5~8 | 4~10 | 7~10 | **H** |
| 5~8 | 4~10 | 1~6 | **M (Medium)** — 팀 검토 후 조치 여부 결정 |
| 5~8 | 1~3 | 모든 값 | **L (Low)** |
| 1~4 | 모든 값 | 모든 값 | **L** |

| AP 등급 | 조치 요건 |
|---|---|
| **H (High)** | 반드시 조치. 담당자 및 완료 목표일 지정. 미조치 시 경영 검토 보고 |
| **M (Medium)** | 다기능 팀 검토 후 조치 여부 결정. 미조치 시 사유 문서화 |
| **L (Low)** | 수용 가능. 기회 발생 시 개선 |

---

### P5. 최적화 (Step 6: Optimization)

**담당**: 다기능 팀  **기한**: AP-H 항목 — 조치 완료 목표일 이내

1. **AP-H 항목 우선 조치**: 아래 수단 중 효과적인 방법 선택
   - **설계 변경**: 공정 파라미터 또는 도면 공차 변경
   - **예방 관리 추가**: 포카요케, 표준화, 작업지침서 개정
   - **검출 방법 추가**: 인라인 센서, 자동 측정 도입, 검사 빈도 증가
2. 조치 완료 후 **S·O·D 재평가** → AP 재산정 (재평가 결과를 PFMEA 양식에 기록)
3. 재평가 후에도 AP-H인 경우: 사유 문서화 및 추가 조치 계획 수립
4. **Control Plan 및 작업지침서(WI) 갱신 지시**: 예방·감지 관리 방법 반영

---

### P6. 결과 문서화 및 유지 관리 (Step 7: Results Documentation)

**담당**: {{dept_quality}}  **기한**: 조치 완료 후 / 변경 발생 시 ≤5영업일 내

1. PFMEA 최종 검토 및 승인 ({{dept_quality}} 서명)
2. **Control Plan과 1:1 연계 확인**:
   - PFMEA 예방 관리 방법 → Control Plan "관리 방법" 열
   - PFMEA 감지 관리 방법 → Control Plan "검사 방법" 열
   - 특별 특성(SC/CC) → Control Plan 특별 특성 표기 및 SPC/100% 검사 적용
3. **고객 특별 특성(SC/CC) PFMEA 반영 여부** 전수 확인
4. PFMEA 개정 이력 관리: 변경 발생 시 ≤5영업일 이내 반영·배포
5. 보관: 해당 제품 수명 기간 + 법적 요건 기간 이상

**PFMEA 갱신 트리거 요약**:

| 사유 | 갱신 범위 | 기한 |
|---|---|---|
| 신규 공정 개발 | 전체 신규 작성 | PPAP 제출 전 |
| 공정 변경 (4M) | 변경 공정 단계 + 영향 공정 | 변경 후 즉시 (≤5일) |
| 고객 클레임 | 해당 공정 + 유사 공정 | 8D 완결 전 |
| 내부 불량 반복 (3회 이상) | 해당 공정 전면 재검토 | 30일 내 |
| 내부·고객 심사 지적 | 해당 항목 | NCR 조치 기한 내 |
| 정기 검토 | 전체 PFMEA | 연 1회 이상 |

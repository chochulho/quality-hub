# 고객불만 및 시정조치 프로세스

**문서번호**: QP-C01  **버전**: {{rev_no}}  **발행일**: {{created_at}}

**회사명**: {{company_name}}  **프로세스 오너**: {{dept_quality}}

---

## 1. 목적

고객으로부터 접수된 불만·클레임·반품을 체계적으로 처리하고 8D(8 Disciplines) 방법론을 통해 근본 원인을 제거하여 동일 불만의 재발을 방지한다. {{cert_target}} 8.2.1.1 및 10.2 요건을 충족한다.

---

## 2. 적용 범위

{{company_name}}이 제공하는 모든 제품·서비스에 대해 고객으로부터 접수되는 불만, 클레임, 반품, VOC(고객의 소리) 전반에 적용한다.

---

| 역할 | 책임 |
|------|------|
| {{dept_sales}} | 고객 불만 최초 접수, 고객 커뮤니케이션, 최종 승인 수령 |
| {{dept_quality}} | 8D 주관, 근본 원인 분석, 시정 조치 유효성 검증 |

| 내부심사팀 | 시정 조치 완결 및 수평 전개 검증 |

## 4. 프로세스 흐름 (SIPOC)

| 구분 | 내용 |
|------|------|
| **SUPPLIER** | 고객(선선자), {{dept_sales}}, 현장 작업자 |
| **INPUT** | 고객불만 접수서, 클레임 현황(불량품·사진·한계선), 고객 요구 납기, 불만 이력 DB |
| **PROCESS** | P1. 불만 접수 → P2. 긴급 봉쇄조치 → P3. 근본원인 분석 → P4. 시정조치 실행 → P5. 유효성 검증 → P6. 수평전개 및 종결 |
| **OUTPUT** | 8D Report, 시정조치 완료 보고서, 고객 최신본 회신, 수평 전개 결과 |
| **CUSTOMER** | 고객사(선선자), {{dept_management}}, 내부심사팀 |

## 5. 세부 절차

### P1. 불만 접수 및 등록 — {{dept_sales}} (당일)

1. 고객으로부터 불만 즉시 고객불만 접수대장에 등록 (QF-C01-01)
2. 불만 유형 분류: 품질문제 / 납기문제 / 서비스문제 / 기타
3. **긴급도 판단 — 긴급(안전·법규·라인정지) → {{dept_quality}} 즉시 보고 / 일반 → 24h 이내 관련 부서 공유**

{{#if cert.iatf}}
> ⚑ IATF 16949 8.5.6 — 고객이 규정한 불만처리 프로세스가 있는 경우 해당 절차를 우선 적용한다.
{{/if}}

---

### P2. 긴급 봉쇄조치 (D3) — {{dept_quality}}, {{dept_production}} (24h 이내)

1. 불량품 현품 또는 사진 확보
2. 고객 재고·창고·생산라인 봉쇄: 해당 Lot/제품 전수 선별 또는 격리
3. 임시대책(Containment) 실행 및 고객에게 48h 이내 회신
4. **D1 팀 구성 / D2 문제 명확화 (5W1H 기술서 작성)**

| 구분 | 내용 |
|------|------|
| 팀 리더 | {{dept_quality}} 품질 엔지니어 |
| 팀 구성 | {{dept_sales}}, {{dept_production}}, {{dept_engineering}} 담당자 포함 |
| 문제 기술 | 누가/무엇이/언제/어디서/얼마나/어떻게 — 5W1H |

---

### P3. 근본원인 분석 (D4–D5) — {{dept_quality}} (3~5일)

1. **5-Why 분석** 실시 — 불량 발생 원인 + 유출 원인 각각 분석
2. 특성요인도(Ishikawa Diagram) 또는 Is/Is Not 분석 병행
3. 발생 원인 및 유출 원인 각각 검증 (실험, 데이터 분석)

{{#if cert.iatf}}
4. **설계기인 → DFMEA 검토** 및 D-FMEA 개정 여부 판단
5. **공정기인 → PFMEA·CP 개정 필요 여부 판단** (QP-D03 연계)
{{/if}}

> 근본 원인이 복합적일 경우 원인별 시정조치 항목을 분리하여 관리한다.

---

### P4. 시정조치 실행 (D6) — {{dept_production}} (합의 납기)

1. 공정변경·원비개선·작업표준서 개정 실행
2. 4M 변경 관리 (Man/Machine/Material/Method) 기록
3. 변경 후 시제품 또는 첫 Lot 특별관리 (강화 검사 실시)

{{#if cert.iatf}}
> ⚑ IATF 16949 — 고객 승인 필요 공정 변경은 반드시 PPAP 재제출 여부를 고객과 협의한다.
{{/if}}

---

### P5. 유효성 검증 (D7) — {{dept_quality}} (30일 이상 모니터링)

1. 동일 품질 불량 재발 여부 30일(또는 3 Lot) 모니터링
2. 모니터링 결과 이상 없을 경우 → 유효성 확인서 발행
3. 이상 발생 시 → P3 근본원인 재분석

| 검증 기준 | 합격 기준 |
|-----------|-----------|
| 재발률 | 0건 |
| 고객 클레임 재접수 | 없음 |
| 강화 검사 합격률 | 100% |

---

### P6. 수평전개 및 종결 (D8) — {{dept_quality}} (완결 후)

1. 유사 공정·제품·라인에 동일 원인 적용 가능성 검토 (수평 전개)
2. FMEA·CP·작업표준·QMS 문서 최종 개정본 확인
3. 8D 최종 보고서 작성 및 고객 제출

{{#if cert.iatf}}
4. 관리계획서(Control Plan) 최신화 확인 (QP-D03 연계)
{{/if}}

5. 고객 최종 승인 수령 → 불만 완결 처리 (QF-C01-01 종결 기록)

## 1. 목적

신제품 및 기존 제품의 중요 변경에 대해 체계적인 설계·개발 프로세스를 수행함으로써 고객 요구사항, 법적·규제 요건을 충족하는 제품을 적시에 출시하고, 설계 단계의 잠재 리스크를 사전에 식별·제거한다.

---

## 2. 적용 범위

다음 경우에 본 절차를 의무 적용한다.

| 적용 사유 | 비고 |
|---|---|
| 신규 제품 개발 (RFQ 수주 이후) | 전체 단계 P1~P6 |
| 기존 제품 중요 설계 변경 (기능·성능·재질 변경) | P1(재검토)~P6 |
| 고객이 APQP/PPAP를 명시 요구하는 경우 | QP-D02 연계 |
| 법적·규제 요건 변경으로 인한 재설계 | P1~P6 |

소규모 외관 변경 또는 포장 변경은 QP-D04(공정 변경)에서 처리한다.

## 5. 세부 절차

### P1. 설계 입력 검토

**담당**: {{dept_engineering}} + {{dept_quality}}  **기한**: 수주 후 5영업일 이내

1. 고객 요구사항(DTS, 도면, 사양서, CSR) 수령 및 검토
2. 모호하거나 상충되는 요건 식별 → 고객에게 서면 명확화 요청 (QF-D01-01 활용)
3. 설계 제약 조건 파악: 치수 공차, 재질, 환경 규제(RoHS·REACH 등), 신뢰성 목표
4. 법적·규제 요건 확인 및 설계 입력 목록(QF-D01-02) 작성
5. 설계 실행 가능성 검토(Feasibility Review): 자체 역량 대비 요구사항 충족 가능성 판단

> **판단 기준**: 설계 입력이 모호하거나 미충족 요건이 있으면 고객 확인 전까지 P2 진행 금지.

---

### P2. 설계 계획 수립

**담당**: {{dept_engineering}}  **기한**: P1 완료 후 3영업일 이내

1. 단계별 일정, 책임자, 마일스톤 수립 → 설계 계획서(QF-D01-03) 작성
2. 검증·확인 방법 및 합격 기준 사전 정의
3. 다기능 팀(CFT: Cross-Functional Team) 구성: 품질·생산·구매 참여
4. {{#if cert.iatf}}APQP 타임라인과 연계 (QP-D02 §P1 참조), Gate 체크리스트 활용{{/if}}
5. 설계 계획서 → {{dept_management}} 승인

---

### P3. 설계 수행

**담당**: {{dept_engineering}}  **기한**: 설계 계획서 일정 준수

1. **개념 설계**: 복수 설계안 도출 → 기술·원가·일정 기준 최적안 선정
2. **상세 설계**: 도면 작성(GD&T 적용), BOM 확정, 재료 규격서 작성
3. **시제품 제작**: {{dept_production}} 협업, 제작 완료 후 치수 측정 기록
4. **DFMEA 작성**: AIAG-VDA 4th Edition(2019) 7단계 방법론 적용
   - 구조 분석 → 기능 분석 → 고장 분석 → 리스크 분석(AP 등급) → 최적화 → 문서화
   - AP-H 항목: 설계 변경 또는 DVP&R 추가 시험으로 반드시 조치
5. **특별 특성(SC/CC) 식별** 및 고객 협의 후 도면·DFMEA 반영
6. **설계 변경 관리**: 변경 발생 시 영향도 분석(QF-D01-04) → {{dept_quality}} + {{dept_engineering}} 공동 승인 후 적용

---

### P4. 설계 검증 (Design Verification)

**담당**: {{dept_engineering}} + {{dept_quality}}  **기한**: 시제품 완성 후

1. **DVP&R(Design Verification Plan & Report)** 작성 — 설계 입력 요건별 검증 방법·합격 기준 명시
2. 시험 실시: 치수 측정, 재질 시험, 기능 시험, 신뢰성 시험(수명·환경 시험 등)
3. 합격 기준: 설계 입력 대비 **100% 충족**
4. 불합격 항목 → 설계 수정 → 재검증 (수정 이력 QF-D01-04에 기록)
5. DVP&R 최종 결과 → {{dept_quality}} 승인, 기록 보관

---

### P5. 유효성 확인 (Design Validation)

**담당**: {{dept_quality}} + 고객  **기한**: 고객 PPAP 승인 전

1. 실제 사용 조건(또는 동등 조건)에서 기능·성능·내구성 확인
2. 현장 파일럿 또는 고객 평가 차량 적용 시험(해당 시)
3. {{#if cert.iatf}}PPAP 제출 전 유효성 확인 완료 필수 (IATF §8.3.4.4){{/if}}
4. 유효성 확인 결과 보고서(QF-D01-05) 작성 → 고객 확인
5. 미충족 시 → 설계 팀과 협의하여 설계 수정 → 재확인

---

### P6. 설계 출력물 관리

**담당**: {{dept_engineering}}  **기한**: 최종 승인 후 즉시

1. 설계 출력물 목록(QF-D01-06) 작성: 도면, BOM, 재료 규격서, DFMEA, DVP&R 등
2. 문서 버전 관리 및 배포 — QP-DC01(문서·기록 관리 프로세스) 연계
3. 설계 기록 보관 기간: 최소 **해당 제품 수명 + 10년** (법적 요건 우선 적용)
4. 설계 출력물 변경 시: QF-D01-04(설계 변경 영향도 분석) → 재승인 후 배포
5. {{#if cert.iatf}}고객 승인(PSW) 기록 포함하여 PPAP 폴더에 통합 관리{{/if}}

# 문서·기록 관리 프로세스

**문서번호**: QP-DC01  **버전**: {{rev_no}}  **발행일**: {{created_at}}

**회사명**: {{company_name}}  **프로세스 오너**: {{dept_quality}}

---

## 1. 목적

{{company_name}}의 QMS 문서 및 기록을 체계적으로 관리하여 최신 문서만 현장에서 사용되도록 하고, 기록의 진본성·가독성·추적성을 보장한다.

## 2. 적용 범위

{{cert_target}} QMS와 관련된 모든 내부 문서(매뉴얼·절차서·지침서·양식)와 기록에 적용한다. 고객 제공 문서(도면·CSR) 포함.

## 3. 책임과 권한

| 역할 | 책임 |
|------|------|
| {{dept_quality}} | 문서 대장 관리, 개정 승인 조율 |
| 각 문서 오너 (프로세스 담당자) | 문서 작성·개정·폐기 요청 |
| 부서장 | 소관 문서 승인 |

## 4. 절차

### 4.1 문서 번호 체계

```
QM-NNN   품질매뉴얼
QP-XXX   프로세스 절차서
QI-XXX   작업지침서
QF-XXX   양식(Form)
KPI-NNN  성과표
MTX-NNN  매트릭스
```

### 4.2 문서 작성 및 검토·승인

1. 문서 작성자는 지정된 양식(QF-DC01-01)으로 초안 작성
2. 관련 부서 검토 → 해당 부서장 승인 → {{dept_quality}} 등록
3. **모든 문서는 발행 전 반드시 승인**을 받아야 한다.

### 4.3 배포 및 현장 관리

1. {{dept_quality}}은 문서 대장(QF-DC01-02)을 최신 상태로 유지한다.
2. 구버전 문서는 현장에서 즉시 수거하고 "폐기" 도장 날인 후 보관 또는 파기한다.
3. 고객 제공 도면은 고객 개정 번호와 동기화하여 관리한다.

### 4.4 문서 개정

1. 개정 요청 → 검토·승인 → 개정 이력 기재 → 재배포
2. 개정 내용은 해당 페이지 여백 표시 또는 개정 이력 기재로 명확히 한다.

### 4.5 기록 관리

| 기록 유형 | 보관 기간 | 보관 위치 |
|---------|---------|---------|
| 내부심사 기록 | 3년 | {{dept_quality}} 서버 |
| 고객 불만·8D | 10년 (고객 CSR 우선) | {{dept_quality}} 서버 |
| 교육훈련 기록 | 재직 기간 + 3년 | {{dept_hr}} 서버 |
| 교정 기록 | 장비 사용 기간 + 1년 | {{dept_quality}} 서버 |
| 생산·검사 기록 | 제품 수명 + 1년 (최소 3년) | {{dept_production}} 서버 |
